# Neo LLVM v0.5.5 Release Notes

Release date: 2026-04-07

## Highlights

- Fixed a real translator nondeterminism bug where identical Wasm could emit different NeoVM scripts because runtime helpers were realized in unstable `HashMap` iteration order.
- Expanded Rust compiler/devpack fuzzing with structured contract generation, differential parity checks, and detached multi-day supervision.
- Confirmed the two saved differential artifacts from the pre-fix bug now replay cleanly, while the ongoing post-fix burn has not produced a new rustc or devpack crash.

## What Changed

### Added

- Added `fuzz_rust_contract` for structured Rust contract compiler/devpack coverage across the end-user Rust -> Wasm -> NeoVM pipeline.
- Added `fuzz_rust_contract_differential` to compile and translate the same generated Rust contract twice and assert Wasm/script/manifest/NEF parity.
- Added `scripts/run_long_fuzz.sh`, `make fuzz-rust-long`, and status reporting suitable for detached multi-day fuzz sessions.
- Added a regression test that decodes the saved differential artifact and verifies deterministic translation in the fuzz support crate.

### Fixed

- Fixed helper realization order in the translator so emitted helper bodies are appended in a stable, sorted key order.
- Fixed the Rust fuzz harness to isolate its temporary Cargo workspace by process id, preventing concurrent long-run sessions from corrupting standalone repros.
- Updated long-run status reporting so it separates artifacts created in the current session from historical artifacts already present on disk.
