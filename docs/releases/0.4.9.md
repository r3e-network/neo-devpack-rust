# Neo LLVM v0.4.9 Release Notes

Release date: 2026-03-22

## Highlights

- First-class `Hash160` and `Hash256` types in the Rust DevPack for type-safe Neo address and hash handling.
- Canonical NEP standard trait definitions (NEP-17/11/24/26/27/29/30/31) enabling compile-time NEP compliance checking.
- Hardened 8 example contracts with state persistence, access control, and event emission.
- DRY refactor of manifest permission deduplication in the translator.

## What Changed

### Added

- `Hash160` and `Hash256` first-class types in `neo-types` with byte-level constructors, hex `Display`, and serde support (`rust-devpack/neo-types/src/hash.rs`).
- NEP-17/11/24/26/27/29/30/31 canonical trait definitions in `rust-devpack/src/standards.rs` for type-safe NEP standard compliance.
- `try_into_i32()`, `try_into_u32()`, `try_into_i64()`, `try_into_u64()` safe integer conversions on `NeoInteger`.

### Changed

- Consolidated manifest permission deduplication logic into a shared `dedup_permissions()` function in `wasm-neovm` (DRY refactor).
- `neo-devpack::codec` now uses `postcard` instead of `bincode`, removing the unmaintained serializer dependency.
- `neo-runtime` contract and crypto helpers use deterministic local implementations compatible with the published `neo-syscalls 0.1.0` surface.
- `neo-test` package verification no longer depends on unpublished `NeoInteger::as_i64_saturating`.
- Local and CI security gates now fail on unmaintained and notice-level `cargo deny` findings.

### Fixed

- Fixed 8 example contracts: added state persistence via storage, public getter methods, `check_witness` access control, and event emission for all state-changing operations.
- Fixed `solana-hello` dev profile: added `panic = "abort"` for `no_std` compatibility on `wasm32-unknown-unknown`.
- Fixed `make quality-check` package-verification phase for `neo-runtime` and `neo-test` tarballs.
- Synced example contract lockfiles with current devpack dependency graph.
