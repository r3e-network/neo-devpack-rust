# Neo LLVM v0.4.5 Release Notes

Release date: 2026-02-07

## Highlights

- Full devpack-style Rust contract syntax support via `#[neo_contract]` + `#[neo_method]` with generated exports from `neo-macros`.
- New DeFi examples: `uniswap-v2`, `staking-rewards`, `timelock-vault`, `flashloan-pool`.
- Expanded local Neo Express smoke validation to deploy/invoke all shipped examples.
- Improved syscall alias coverage and regression testing for Neo N3 canonical `System.*` descriptors.

## New and Updated Examples

- Added and validated:
  - `uniswap-v2`
  - `staking-rewards`
  - `timelock-vault`
  - `flashloan-pool`
- Advanced templates (`multisig-wallet`, `escrow`, `crowdfunding`, `governance-dao`, `oracle-consumer`, `nft-marketplace`, `move-coin`) were standardized to devpack syntax and made deploy-safe for local Neo Express smoke runs.

## Syscall and Native Contract Support

- Translator-side `neo` import aliasing now includes canonical coverage for generated Neo N3 engine syscalls and generated snake_case aliases.
- Added coverage for edge alias forms used by real contracts, including runtime hash getters and storage readonly-context variants.
- `System.Contract.CallNative` remains mapped, with runtime semantics matching Neo N3 (direct use is native-contract restricted by the VM).

## Verification

The following commands pass in the release workspace:

- `make examples`
- `make smoke-neoxp`
- `make test`
- `make test-cross-chain`

## Notes

- Optional `c-hello` build is skipped when `wasm-ld` is unavailable.
- This release emphasizes local production-readiness validation through deterministic deploy/invoke smoke tests on Neo Express.
