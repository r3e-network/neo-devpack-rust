# Neo LLVM v0.5.4 Release Notes

Release date: 2026-04-04

## Highlights

- Fixed manifest rendering so `storage` and `payable` contract features survive into emitted manifests.
- Hardened validation APIs around syscall descriptor parsing and alignment construction.
- Corrected release tooling so version consistency checks reflect the repository's split-version policy.

## What Changed

### Added

- Added `Alignment::try_new(...)` for fallible alignment construction in public validation paths.
- Added regression coverage for manifest feature retention, canonical syscall descriptors, and invalid alignment inputs.

### Changed

- `SyscallDescriptor` validation now enforces canonical `Root.Category.Method` identifiers.
- Refreshed README and documentation for manifest generation, cross-chain status, and current translator behavior.

### Fixed

- Fixed manifest rendering so `storage` and `payable` flags are preserved in emitted contract manifests.
- Fixed `scripts/check_versions.sh` so it checks package-local `version.workspace = true` accurately instead of matching `rust-version.workspace = true`.

## Verification

The following commands pass for this release:

- `bash scripts/check_versions.sh`
- `cargo fmt --all --check`
- `cargo test --workspace`
- `cargo clippy --workspace --all-targets -- -D warnings`
