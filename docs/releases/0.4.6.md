# Neo LLVM v0.4.6 Release Notes

Release date: 2026-02-11

## Highlights

- Hardened `#[neo_contract]` export wrappers for `NeoResult<NeoInteger>`, `NeoResult<NeoBoolean>`, and `NeoResult<()>` with deterministic per-method `<MethodName>LastError` status exports.
- Tightened WASM active segment parsing to reject invalid negative offsets instead of allowing wraparound behavior.
- Enforced strict 20-byte `Hash160` argument validation in syscall wrapper decoding.
- Made default API constructors safer by normalizing empty contract names to `Contract` in non-fallible constructors.

## Security and Correctness Improvements

- Data segment and element segment active offsets are now validated as non-negative before conversion.
- Table element function references now reject indices that exceed NeoVM table index bounds.
- Wrapper error signaling no longer depends on panic paths for `NeoResult<()>` exports.

## Compatibility Notes

- Existing generated contract exports remain callable; failing calls now expose explicit status via `<MethodName>LastError`.
- `TranslationBuilder::try_new` behavior remains unchanged and still rejects empty names.
