# Neo LLVM v0.4.8 Release Notes

Release date: 2026-02-16

## Highlights

- Hardened Neo N3 syscall translation parity across devpack and translator descriptor/hash/alias surfaces.
- Strengthened import hardening for `opcode`, `syscall`, and `env` boundaries with fail-closed behavior and improved diagnostics.
- Completed fixed-width opcode immediate coverage for `PUSHINT128` and `PUSHINT256`.

## What Changed

- Updated workspace package version metadata to `0.4.8`.
- Updated `wasm-neovm` dependency version pin in workspace dependencies to `0.4.8`.
- Added conformance tests:
  - syscall parity tests across `neo-syscalls` and translator descriptors.
  - syscall reference vectors for critical Neo syscall hashes and alias lowering.
  - opcode fixed-width operand support guard, including 128-bit and 256-bit immediate paths.
- Hardened adapters so `syscall` imports resolve extended Neo descriptors consistently across Neo/Move/Solana source chains.
- Hardened opcode import translation:
  - strict void-signature and arity validation.
  - explicit 16-byte and 32-byte immediate encoding support.
  - deterministic sign extension for 256-bit immediates.
- Expanded env import alias mapping to accept `__memcpy`, `__memmove`, and `__memset`.
- Improved import diagnostics to preserve original import spelling and include full `module::name` context.

- `cargo test --workspace --all-targets --all-features --no-fail-fast`
