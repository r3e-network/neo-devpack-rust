neo-llvm is a production-ready Rust toolchain for compiling [Neo N3](https://neo.org/) smart contracts. It takes standard Rust code, compiles it to WebAssembly, then translates the Wasm bytecode into NeoVM instructions — producing the NEF binary and manifest JSON that Neo N3 nodes expect.

## Compilation Pipeline

```
Rust Contract → cargo build (wasm32) → Wasm Module → wasm-neovm → NEF + Manifest
```

1. You write a smart contract in Rust using the `rust-devpack` SDK.
2. `cargo build` compiles it to a `.wasm` binary targeting `wasm32-unknown-unknown`.
3. The `wasm-neovm` translator reads the Wasm module and emits NeoVM opcodes.
4. The output is a `.nef` file (the on-chain bytecode) and a `.manifest.json` file (the contract metadata).

## Why Rust?

Neo N3 contracts are traditionally written in C#. Rust offers a different set of trade-offs:

| | Rust (neo-llvm) | C# (neo-devpack-dotnet) |
|---|---|---|
| Memory safety | Compile-time guarantees via ownership | Garbage-collected runtime |
| Ecosystem | Cargo, crates.io, mature tooling | NuGet, .NET SDK |
| Performance | Zero-cost abstractions | JIT-compiled |
| Learning curve | Steeper, but enforces correctness | Gentler for .NET developers |
| Contract size | Typically smaller Wasm output | Managed IL output |

Both toolchains produce valid NEF bytecode. Choose the one that fits your team and project.
