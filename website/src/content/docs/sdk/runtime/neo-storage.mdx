`NeoStorage` provides CRUD access to a contract's persistent key-value store on the Neo N3 blockchain. Every deployed contract has its own isolated storage area. Data written here survives across invocations and is the primary mechanism for maintaining contract state.

All operations require a `NeoStorageContext` handle obtained from the runtime. See [Storage Context](/sdk/types/storage-context/) for details on read-write vs read-only contexts.

```rust
use neo_devpack::prelude::*;

#[neo_contract]
impl MyToken {
    #[neo_method(safe)]
    pub fn balance_of(account: ScriptHash) -> i64 {
        let ctx = NeoStorage::get_read_only_context();
        let key = storage_key("balance", &account);
        NeoStorage::get_integer(&ctx, &key)
    }
}
```

## get

Reads a value from persistent storage by key. Returns the raw bytes stored at that key, or an empty byte array if the key does not exist.

**Syscall:** `System.Storage.Get`

```rust
use neo_devpack::prelude::*;

fn read_owner(ctx: &NeoStorageContext) -> ScriptHash {
    let data = NeoStorage::get(ctx, b"owner");
    ScriptHash::from_bytes(&data)
}
```

| Item | Detail |
|------|--------|
| Parameters | `ctx: &NeoStorageContext`, `key: &[u8]` |
| Returns | `Vec<u8>` — raw bytes, or empty if key absent |
| GAS cost | Scales with value size |

### Typed convenience methods

`NeoStorage` provides typed wrappers that deserialize the stored bytes into common types:

| Method | Returns | Description |
|--------|---------|-------------|
| `get(ctx, key)` | `Vec<u8>` | Raw bytes |
| `get_integer(ctx, key)` | `i64` | Interpret bytes as a little-endian integer; returns `0` if absent |
| `get_string(ctx, key)` | `NeoString` | Interpret bytes as UTF-8 string; returns empty string if absent |
| `get_bool(ctx, key)` | `bool` | Interpret bytes as boolean; returns `false` if absent |

```rust
use neo_devpack::prelude::*;

fn read_config(ctx: &NeoStorageContext) {
    let name: NeoString = NeoStorage::get_string(ctx, b"config:name");
    let decimals: i64    = NeoStorage::get_integer(ctx, b"config:decimals");
    let paused: bool     = NeoStorage::get_bool(ctx, b"config:paused");
}
```

## put

Writes a value to persistent storage. If the key already exists, the value is overwritten. Writing an empty byte array is valid and distinct from deleting the key.

**Syscall:** `System.Storage.Put`

```rust
use neo_devpack::prelude::*;

fn set_owner(ctx: &NeoStorageContext, owner: &ScriptHash) {
    NeoStorage::put(ctx, b"owner", owner.as_bytes());
}
```

| Item | Detail |
|------|--------|
| Parameters | `ctx: &NeoStorageContext`, `key: &[u8]`, `value: &[u8]` |
| Returns | `()` |
| GAS cost | Scales with key + value size; first write to a key costs more than updates |

### Typed convenience methods

| Method | Value type | Description |
|--------|-----------|-------------|
| `put(ctx, key, value)` | `&[u8]` | Raw bytes |
| `put_integer(ctx, key, value)` | `i64` | Store as little-endian bytes |
| `put_string(ctx, key, value)` | `&str` | Store as UTF-8 bytes |
| `put_bool(ctx, key, value)` | `bool` | Store as single byte (`0x01` / `0x00`) |

```rust
use neo_devpack::prelude::*;

fn initialize(ctx: &NeoStorageContext) {
    NeoStorage::put_string(ctx, b"config:name", "MyToken");
    NeoStorage::put_integer(ctx, b"config:decimals", 8);
    NeoStorage::put_bool(ctx, b"config:paused", false);
}
```

<Aside type="caution">
  `NeoStorage::put` requires a read-write context. Passing a read-only context will cause the NeoVM to fault and the transaction to revert.
</Aside>

## delete

Removes a key and its value from persistent storage. If the key does not exist, this is a no-op. Deleting a key refunds a portion of the GAS originally spent to store it.

**Syscall:** `System.Storage.Delete`

```rust
use neo_devpack::prelude::*;

fn remove_approval(ctx: &NeoStorageContext, owner: &ScriptHash, spender: &ScriptHash) {
    let key = storage_key_2("approve", owner, spender);
    NeoStorage::delete(ctx, &key);
}
```

| Item | Detail |
|------|--------|
| Parameters | `ctx: &NeoStorageContext`, `key: &[u8]` |
| Returns | `()` |
| GAS cost | Low; partial refund of original storage cost |

## find

Iterates over all storage entries whose keys begin with the given prefix. Returns a `NeoIterator` that yields key-value pairs in lexicographic key order.

**Syscall:** `System.Storage.Find`

```rust
use neo_devpack::prelude::*;

#[neo_method(safe)]
pub fn all_balances() -> NeoArray<(ScriptHash, i64)> {
    let ctx = NeoStorage::get_read_only_context();
    let iter = NeoStorage::find(&ctx, b"balance:");
    let mut result = NeoArray::new();

    while iter.next() {
        let key = iter.key();       // full key including prefix
        let value = iter.value();   // raw bytes
        let account = ScriptHash::from_bytes(&key[8..]); // strip "balance:" prefix
        let amount = bytes_to_i64(&value);
        result.push((account, amount));
    }

    result
}
```

| Item | Detail |
|------|--------|
| Parameters | `ctx: &NeoStorageContext`, `prefix: &[u8]` |
| Returns | `NeoIterator` — lazy iterator over matching key-value pairs |
| GAS cost | Base cost + per-entry cost as you iterate |

<Aside type="tip">
  `find` returns a lazy iterator. Entries are fetched from storage one at a time as you call `next()`. This is efficient for large datasets — you only pay GAS for the entries you actually read.
</Aside>

## Storage context management

Before performing any storage operation, you need a context handle. `NeoStorage` provides two methods to obtain one:

| Method | Syscall | Permissions |
|--------|---------|-------------|
| `NeoStorage::get_context()` | `System.Storage.GetContext` | Read + Write |
| `NeoStorage::get_read_only_context()` | `System.Storage.GetReadOnlyContext` | Read only |

```rust
use neo_devpack::prelude::*;

#[neo_method]
pub fn update_balance(account: ScriptHash, amount: i64) {
    // Read-write context for modifications
    let ctx = NeoStorage::get_context();
    let key = storage_key("balance", &account);
    NeoStorage::put_integer(&ctx, &key, amount);
}

#[neo_method(safe)]
pub fn balance_of(account: ScriptHash) -> i64 {
    // Read-only context for queries
    let ctx = NeoStorage::get_read_only_context();
    let key = storage_key("balance", &account);
    NeoStorage::get_integer(&ctx, &key)
}
```

### When to use read-only context

- In `safe` methods that only read state
- When passing a context to an external contract (prevents unauthorized writes)
- As a defensive measure in helper functions that should not modify storage

## Key design patterns

### Prefix namespacing

Use a consistent prefix scheme to organize storage keys. This prevents collisions between different data types and enables efficient prefix iteration with `find`.

```rust
// Recommended key layout for a token contract:
//
//   "owner"                          → ScriptHash (single value)
//   "config:name"                    → String
//   "config:symbol"                  → String
//   "config:decimals"                → i64
//   "balance:<script_hash>"          → i64
//   "approve:<owner>:<spender>"      → i64
//   "meta:total_supply"              → i64

fn storage_key(prefix: &str, account: &ScriptHash) -> Vec<u8> {
    let mut key = Vec::with_capacity(prefix.len() + 1 + 20);
    key.extend_from_slice(prefix.as_bytes());
    key.push(b':');
    key.extend_from_slice(account.as_bytes());
    key
}
```

### Composite keys

For multi-dimensional lookups (e.g., allowances keyed by owner + spender), concatenate the components with a delimiter:

```rust
fn approval_key(owner: &ScriptHash, spender: &ScriptHash) -> Vec<u8> {
    let mut key = Vec::with_capacity(8 + 20 + 1 + 20);
    key.extend_from_slice(b"approve:");
    key.extend_from_slice(owner.as_bytes());
    key.push(b':');
    key.extend_from_slice(spender.as_bytes());
    key
}
```

### Enumeration via prefix

Design keys so that related entries share a prefix, enabling batch reads with `find`:

```rust
use neo_devpack::prelude::*;

/// Count all token holders by iterating the "balance:" prefix.
#[neo_method(safe)]
pub fn holder_count() -> i64 {
    let ctx = NeoStorage::get_read_only_context();
    let iter = NeoStorage::find(&ctx, b"balance:");
    let mut count: i64 = 0;

    while iter.next() {
        let balance = bytes_to_i64(&iter.value());
        if balance > 0 {
            count += 1;
        }
    }

    count
}
```

## Complete example

A minimal key-value registry contract demonstrating all four operations:

```rust
use neo_devpack::prelude::*;

neo_manifest_overlay!(r#"{
    "name": "KeyValueStore",
    "features": { "storage": true }
}"#);

#[neo_contract]
pub struct KeyValueStore;

#[neo_contract]
impl KeyValueStore {
    pub fn new() -> Self { Self }

    /// Store a value under the given key.
    #[neo_method]
    pub fn set(key: &[u8], value: &[u8]) -> bool {
        let ctx = NeoStorage::get_context();
        let full_key = prefixed_key(b"data:", key);
        NeoStorage::put(&ctx, &full_key, value);
        NeoRuntime::log("key stored");
        true
    }

    /// Read a value by key.
    #[neo_method(safe)]
    pub fn get(key: &[u8]) -> Vec<u8> {
        let ctx = NeoStorage::get_read_only_context();
        let full_key = prefixed_key(b"data:", key);
        NeoStorage::get(&ctx, &full_key)
    }

    /// Delete a key.
    #[neo_method]
    pub fn remove(key: &[u8]) -> bool {
        let ctx = NeoStorage::get_context();
        let full_key = prefixed_key(b"data:", key);
        NeoStorage::delete(&ctx, &full_key);
        true
    }

    /// List all keys with the given prefix.
    #[neo_method(safe)]
    pub fn list(prefix: &[u8]) -> NeoArray<Vec<u8>> {
        let ctx = NeoStorage::get_read_only_context();
        let full_prefix = prefixed_key(b"data:", prefix);
        let iter = NeoStorage::find(&ctx, &full_prefix);
        let mut keys = NeoArray::new();

        while iter.next() {
            keys.push(iter.key());
        }

        keys
    }
}

fn prefixed_key(prefix: &[u8], key: &[u8]) -> Vec<u8> {
    let mut result = Vec::with_capacity(prefix.len() + key.len());
    result.extend_from_slice(prefix);
    result.extend_from_slice(key);
    result
}
```
