This is the minimal starting point for any neo-devpack-rust contract. It declares one safe (read-only) method, returns a constant, and demonstrates the core macro pattern that every contract builds on.

## What You Will Learn

- The `neo_manifest_overlay!` macro for embedding contract metadata
- The `#[neo_contract]` struct and impl pattern
- Marking methods as `safe` (read-only) with `#[neo_method(safe)]`
- Building and translating a contract to NeoVM bytecode

## Full Source

```rust
// contracts/hello-world/src/lib.rs
use neo_devpack::prelude::*;

neo_manifest_overlay!(
    r#"{
    "name": "HelloWorld"
}"#
);

const HELLO_RESULT: i64 = 42;

#[neo_contract]
pub struct HelloWorldContract;

#[neo_contract]
impl HelloWorldContract {
    pub fn new() -> Self {
        Self
    }

    #[neo_method(safe)]
    pub fn hello() -> i64 {
        HELLO_RESULT
    }
}

impl Default for HelloWorldContract {
    fn default() -> Self {
        Self::new()
    }
}
```

## Key Patterns

`neo_manifest_overlay!` — Embeds a JSON fragment that the translator merges into the final `.manifest.json`. At minimum, set the contract `"name"`. Standards, permissions, and trust settings can be added here for more complex contracts.

`#[neo_contract]` — Applied to both the struct declaration and its `impl` block. This tells the macro system to generate the Wasm exports and NeoVM entry points for every `#[neo_method]` inside the impl.

`#[neo_method(safe)]` — Marks `hello()` as a read-only method. Safe methods can be called without spending GAS and cannot modify on-chain storage. In the generated manifest, this method will have `"safe": true`.

`Default` impl — Required by the macro expansion. Every `#[neo_contract]` struct needs a `Default` implementation.

## Build and Run

<Steps>

1. **Compile to Wasm**

   ```bash
   cargo build --manifest-path contracts/hello-world/Cargo.toml \
     --release --target wasm32-unknown-unknown
   ```

2. **Translate to NeoVM**

   ```bash
   make hello-world
   ```

   This produces two files in `build/`:
   - `HelloWorld.nef` — the NeoVM bytecode
   - `HelloWorld.manifest.json` — the contract manifest

3. **Run the tests**

   ```bash
   cargo test --manifest-path contracts/hello-world/Cargo.toml
   ```

</Steps>

## What to Look For

In the generated `HelloWorld.manifest.json`, verify:

- The `name` field is `"HelloWorld"`
- The `abi.methods` array contains a single entry for `hello` with `"safe": true`
- The method's return type is `Integer`

<Aside type="tip">
This contract has zero storage access and zero cross-contract calls, making it the fastest possible contract to compile and the best target for verifying your toolchain installation.
</Aside>
