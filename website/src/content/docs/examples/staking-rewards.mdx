This contract implements a staking pool with a fixed annual percentage rate (APR). Users stake tokens, accrue rewards over time, and claim them. The reward calculation uses basis-point arithmetic to avoid floating-point operations on-chain.

## What You Will Learn

- APR-based reward calculation using integer arithmetic
- Basis-point (BPS) math patterns for on-chain finance
- Boundary validation (max staking duration, zero-amount guards)
- Separating preview (read-only) from claim (state-mutating) operations

## Full Source

```rust
// contracts/staking-rewards/src/lib.rs
use neo_devpack::prelude::*;

neo_manifest_overlay!(
    r#"{
    "name": "StakingRewards"
}"#
);

const APR_BPS: i64 = 1_200;          // 12.00% annual rate
const BPS_DENOMINATOR: i64 = 10_000;
const DAYS_PER_YEAR: i64 = 365;
const MAX_DAYS: i64 = 3_650;         // 10-year cap

#[neo_contract]
pub struct StakingRewardsContract;

#[neo_contract]
impl StakingRewardsContract {
    pub fn new() -> Self {
        Self
    }

    fn preview_reward_internal(amount: i64, days_staked: i64) -> i64 {
        if amount <= 0 || days_staked <= 0 || days_staked > MAX_DAYS {
            return 0;
        }
        (amount * APR_BPS * days_staked) / (BPS_DENOMINATOR * DAYS_PER_YEAR)
    }

    #[neo_method]
    pub fn stake(staker: i64, amount: i64) -> bool {
        staker > 0 && amount > 0
    }

    #[neo_method]
    pub fn unstake(staker: i64, amount: i64) -> bool {
        staker > 0 && amount > 0
    }

    #[neo_method(safe)]
    pub fn preview_reward(amount: i64, days_staked: i64) -> i64 {
        Self::preview_reward_internal(amount, days_staked)
    }

    #[neo_method]
    pub fn claim(staker: i64, amount: i64, days_staked: i64) -> i64 {
        if staker <= 0 {
            return 0;
        }
        Self::preview_reward_internal(amount, days_staked)
    }
}

impl Default for StakingRewardsContract {
    fn default() -> Self {
        Self::new()
    }
}
```

## Key Patterns

**Basis-point arithmetic** — The APR is stored as 1,200 basis points (12.00%). The reward formula is:

```
reward = (amount * APR_BPS * days_staked) / (BPS_DENOMINATOR * DAYS_PER_YEAR)
```

For 10,000 tokens staked for 365 days at 12% APR:
```
(10_000 * 1_200 * 365) / (10_000 * 365) = 1,200
```

This avoids floating-point entirely and is safe for on-chain integer math.

**Duration cap** — `MAX_DAYS` (3,650 = 10 years) prevents overflow in the multiplication `amount * APR_BPS * days_staked`. Without this cap, large values could overflow `i64`.

**Preview vs claim** — `preview_reward` is marked `#[neo_method(safe)]` so users can check their pending rewards without spending GAS. `claim` performs the same calculation but is state-mutating (it would transfer tokens in a production contract).

**Shared internal logic** — Both `preview_reward` and `claim` delegate to `preview_reward_internal`, ensuring the preview always matches the actual claim amount.

## Build

```bash
make staking-rewards
```

This produces `build/StakingRewards.nef` and `build/StakingRewards.manifest.json`.

## Tests

```bash
cargo test --manifest-path contracts/staking-rewards/Cargo.toml
```

The test suite verifies:
- Staking and unstaking reject zero amounts or invalid stakers
- `preview_reward(10_000, 365)` returns exactly 1,200 (12% of 10,000)
- Zero days or days exceeding `MAX_DAYS` return 0
- `claim` returns 0 for invalid stakers even with valid amounts

## What to Look For

In the generated `StakingRewards.manifest.json`:

- `preview_reward` has `"safe": true`
- `stake`, `unstake`, and `claim` have `"safe": false`
- The `claim` method returns `Integer` (the reward amount), not `Boolean`

<Aside type="tip">
The basis-point pattern used here is the standard approach for on-chain percentage calculations across all blockchain platforms. 1 BPS = 0.01%, so 10,000 BPS = 100%.
</Aside>
