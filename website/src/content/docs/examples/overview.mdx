## Building Examples

Every example follows the same two-step build process: compile to Wasm, then translate to NeoVM.

### Single contract

```bash
# Step 1: Compile Rust → Wasm
cargo build --manifest-path contracts/<name>/Cargo.toml \
  --release --target wasm32-unknown-unknown

# Step 2: Translate Wasm → NeoVM (.nef + manifest)
cargo run --manifest-path wasm-neovm/Cargo.toml --quiet -- \
  contracts/<name>/target/wasm32-unknown-unknown/release/<crate_name>.wasm \
  --output build/
```

### Using the Makefile

The top-level `Makefile` wraps both steps for every contract:

```bash
# Build a single example
make hello-world
make nep17-token
make governance-dao

# Build all examples at once
make examples
```

Each target produces a `.nef` file and a `.manifest.json` file in the `build/` directory.

### Running tests

Every contract includes unit tests that run natively (no Wasm target needed):

```bash
# Test a single contract
cargo test --manifest-path contracts/<name>/Cargo.toml

# Test all contracts
cargo test --workspace
```

## Common Patterns

All example contracts share a consistent structure:

```rust
use neo_devpack::prelude::*;

neo_manifest_overlay!(r#"{
    "name": "ContractName"
}"#);

#[neo_contract]
pub struct MyContract;

#[neo_contract]
impl MyContract {
    pub fn new() -> Self { Self }

    #[neo_method(safe)]   // read-only, no state changes
    pub fn query() -> i64 { 42 }

    #[neo_method]          // state-mutating
    pub fn mutate(x: i64) -> bool { x > 0 }

    #[neo_method(name = "onNEP17Payment")]  // custom ABI name
    pub fn on_nep17_payment(_from: i64, _amount: i64, _data: i64) {}
}
```
