# Sample Contracts

This directory provides ready-to-build contracts that exercise the Wasm → NeoVM
pipeline. Most samples are Rust crates that compile to `cdylib` Wasm modules,
and a standalone C example is included to demonstrate the clang workflow.

Rust samples now use impl-level `#[neo_contract]` with `#[neo_method]` declarations; exported ABI entrypoints are auto-generated by `neo-macros`, so handwritten `pub extern "C"` wrappers are no longer required.

| Contract | Description | Entry Points | Notes |
| --- | --- | --- | --- |
| `hello-world` | Minimal example returning a constant value. | `hello()` | Marks `hello` safe via `#[neo_safe]`. |
| `nep17-token` | Toy NEP‑17-shaped sample with fixed balances and witness-gated `init`/`transfer` validation. | `init(owner, amount)`, `totalSupply()`, `balanceOf(account)`, `transfer(from, to, amount)`, `onNEP17Payment(from, amount, data)` | Uses synthetic `i64` account IDs for examples; it does not persist balances or emit full production NEP‑17 transfer flows. |
| `constant-product` | Constant-product AMM with storage-backed reserves, swap fees, and swap events. | `init(initial_x, initial_y)`, `getReserves()`, `quote(amount_in)`, `swap(trader, amount_in)` | Declares query methods safe via `#[neo_safe]`. |
| `nep11-nft` | Minimal NEP‑11 NFT that supports minting, owner lookup, and transfers. | `mint(owner, token_id)`, `totalSupply()`, `balanceOf(owner)`, `ownerOf(token_id)`, `transfer(from, to, token_id)` | View methods use `#[neo_safe]`; emits NEP‑11 transfer events. |
| `uniswap-v2` | Uniswap V2-style constant-product AMM quote/swap example. | `addLiquidity(amount_0, amount_1)`, `getReserves()`, `quote(amount_in)`, `swapExactTokensForTokens(amount_in, min_out)` | Applies a 0.3% fee model with ratio/slippage validations for deploy-safe smoke coverage. |
| `staking-rewards` | Staking rewards pool example with APR-based reward preview. | `stake(staker, amount)`, `unstake(staker, amount)`, `previewReward(amount, days_staked)`, `claim(staker, amount, days_staked)` | Includes reward horizon validation and deterministic claim/reward checks. |
| `timelock-vault` | Timelock/vesting vault example with witness checks and runtime-backed release maturity. | `queueRelease(caller_id, beneficiary, amount, unlock_time)`, `isMature(unlock_time, current_time)`, `release(vault_id, caller_id, current_time)` | Uses `System.Runtime.GetTime` when available for enforcement; the safe helper keeps a fallback argument for host-side smoke tests. |
| `flashloan-pool` | Flashloan pool example (ERC-3156 style concepts adapted for Neo). | `maxFlashLoan()`, `flashFee(amount)`, `flashLoan(borrower, amount)`, `repay(amount, repaid_amount)` | Validates fee/repayment invariants with bounded liquidity constraints for local smoke testing. |
| `multisig-wallet` | Configurable multisig wallet with owner-witness-gated proposals and contract calls. | `configure(owner1, owner2, owner3, owner_count, threshold)`, `propose(proposer_id, target_id, method_id, arg_data)`, `approve(proposal_id, signer_id)`, `execute(proposal_id, executor_id)`, `getConfig()` | Sample APIs use compact `i64` owner IDs; configuration, proposal, approval, and execution now require matching runtime witnesses. |
| `escrow` | Arbiter-controlled NEP‑17 escrow with manifest-backed events. | `configure(...)`, `release(...)`, `refund(...)`, `onNEP17Payment(from, amount, data)`, `getState()` | `getState` marked safe; funds released via `NeoContractRuntime::call`. |
| `crowdfunding` | Campaign-style crowdfunding with NEP‑17 contributions and refunds. | `configure(...)`, `finalize(current_time)`, `claimRefund(...)`, `onNEP17Payment(from, amount, data)`, `getCampaign()`, `contributionOf(address)` | Safe views for campaign state/contributions; contributions tracked per address. |
| `governance-dao` | Stake-weighted governance with proposals, voting, execution, and staking. | `configure(...)`, `propose(...)`, `vote(...)`, `execute(proposal_id)`, `unstake(...)`, `onNEP17Payment(from, amount, data)`, `stakeOf(address)` | `stakeOf` is safe; proposal execution is represented with compact integer IDs in this smoke-oriented sample. |
| `oracle-consumer` | Oracle request/response manager with owner-gated requests and caller-verified oracle callbacks. | `configure(owner_id, oracle_id)`, `request(url_id, filter_id, user_data_id)`, `onOracleResponse(request_id, code, data_id)`, `getConfig()`, `getResponse(id)`, `lastRequestId()` | Stores compact integer response fields; callback acceptance is tied to the configured calling contract rather than a user-supplied identity. |
| `nft-marketplace` | Custodial NEP‑11 marketplace settling sales with NEP‑17 payments. | `createListing(...)`, `cancelListing(...)`, `onNEP11Payment(from, token_id, amount, data)`, `onNEP17Payment(from, amount, data)`, `getListing(id)` | Listings persisted in storage; safe listing lookup for tooling. |
| `c-hello` | Minimal C contract compiled via clang helper. | `sum(a, b)`, `version()`, `clamp_add(value, delta, max)` | Uses `manifest.overlay.json` to declare ABI metadata and safe flags. |

## Building

Run the Makefile target to compile all examples and translate them to NEF/manifest files:

```bash
make examples
```

Artifacts appear in the repository-level `build/` directory. To regenerate a specific contract,
invoke its target (e.g., `make nep17-token`).

## Unit Testing

Run unit tests across all Rust sample contracts with:

```bash
make test-contracts
```

This target auto-discovers and executes `cargo test` for every Rust contract
crate under `contracts/`, including `solana-hello` in host test mode. It also
checks that each contract crate defines test markers to prevent silent gaps.

## Deployment

Refer to `scripts/neoexpress_deploy.sh` and [`docs/neoexpress-integration.md`](../docs/neoexpress-integration.md)
for instructions on deploying the generated artifacts to a Neo Express instance.
